1.3 — normalizeHex(input: string): string | null

Trim + lowercase.
Accept #rrggbb (return as-is lowercase) and #rgb (expand each nibble: f0a → #ff00aa).
Return null for anything else.
1.4 — colorDistance(a: string, b: string): number

Parse to [r,g,b] via parseInt(h.slice(n,n+2), 16).
Return Math.sqrt((Δr)² + (Δg)² + (Δb)²).
Max possible: 441.67. Works fine for the MIN_SEP of 18 used at palette-build time.
1.5 — isTaken(hex: string, taken: ReadonlySet<string>): boolean

Returns taken.has(hex) || RESERVED_COLORS.has(hex).
hex must already be normalized (caller's responsibility).
1.6 — suggestAlternative(desired: string, taken: ReadonlySet<string>): string

Walk outward from desired by rotating hue in steps of ±12°, then ±24°, ±36°… (interleaved ±). At each candidate, also try lightness ±8%.
For each candidate: normalize, check not taken, not reserved, colorDistance(candidate, desired) >= 22 (visibly "a bit apart" per requirement).
Cap walk at 120 iterations; if still no candidate found, fall through to pickSuggestedPalette(1, taken)[0] (guaranteed to find something from the 100).
Return the first matching candidate.
Implementation: use HSL math — hexToHsl, step hue, hslToHex.
1.7 — pickSuggestedPalette(count: number, taken: ReadonlySet<string>): string[]

Iterate BASE_PALETTE; collect entries where !isTaken(entry, taken).
If fewer than count entries remain, generate additional colours by rotating hue evenly until count are collected.
Return first count. Called by the gateway to produce the 6 swatches.
1.8 — assignUniqueColor(seed: string, taken: ReadonlySet<string>): string

For AI / fallback assignment. Hash seed (FNV-32), map to a BASE_PALETTE index, walk forward until !isTaken. If all 100 exhausted, generate via HSL rotation.
Deterministic: same seed + same taken → same output.

Phase 4 — Gateway: buildTakenColorSet helper (inside createGatewayApp)
4.1 Add a private helper inside createGatewayApp (not exported):

const buildTakenColorSet = async (excludePlayerId: string): Promise<Set<string>> => {
  const taken = new Set<string>(RESERVED_COLORS);
  // 1. stored profiles
  for (const profile of await profileStore.listAllNamed()) {
    if (profile.playerId === excludePlayerId) continue;
    const n = normalizeHex(profile.tileColor ?? "");
    if (n) taken.add(n);
  }
  // 2. live overrides (supersede stored for active sessions)
  for (const [pid, override] of profileOverrides.entries()) {
    if (pid === excludePlayerId) continue;
    const n = normalizeHex(override.tileColor ?? "");
    if (n) taken.add(n);
  }
  // 3. seeded AI colors (profileOverrides covers them after Phase 5, but belt + braces)
  for (const aiId of seededAiPlayerIds) {
    if (aiId === excludePlayerId) continue;
    const override = profileOverrides.get(aiId);
    const n = normalizeHex(override?.tileColor ?? "");
    if (n) taken.add(n);
  }
  return taken;
};

Phase 6 — Gateway: AI empire color seeding
At the AI seed block (gateway-app.ts lines 960–984), after seededAiPlayerIds is built, add:

// Assign each AI a unique colour from BASE_PALETTE and register it
// in profileOverrides so the taken-set sees it immediately.
const aiTaken = new Set<string>(RESERVED_COLORS);
for (const aiId of [...seededAiPlayerIds].sort()) { // sort for determinism
  const color = assignUniqueColor(aiId, aiTaken);
  aiTaken.add(color);
  // Write to profileStore so it persists across restarts
  await profileStore.setTileColor(aiId, color);
  profileOverrides.upsert(aiId, { tileColor: color });
}
This runs once at startup. On restart the AI ids already have stored colors, so assignUniqueColor will see the same taken set and return the same result (deterministic) — effectively idempotent.
